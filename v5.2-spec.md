# Council v5.2 Spec — Verdict Accuracy Fixes

## Status: READY TO IMPLEMENT
v5.1 test results: 7/10 accuracy, 56pt spread. 3 issues to fix.

## Fix 1: WEAK_VERDICT_BUG (high priority)
**Problem**: Test 1 got 3 penalties applied but verdict "risky" instead of "weak". Model ignored MUST weak rule.
**Root cause**: Prompt says "MUST be weak" but model overrides when baseScore is high (61).
**Fix in `strategist.ts`**:
- Add explicit instruction at TOP of verdict section: `"CRITICAL: If 3 penalties are applied, verdict is ALWAYS 'weak' — no exceptions, regardless of baseScore."`
- Add to RULES section: `"VERIFICATION STEP: Before outputting verdict, count applied penalties. If count == 3, set verdict = 'weak'. This overrides all other rules."`
- In concise prompt: same reinforcement

## Fix 2: STRONG_UNREACHABLE (medium priority)
**Problem**: Test 6 ($2M seed, $50K MRR, 200 merchants, 15% MoM) scored 71 — "promising" not "strong". Strong requires >= 80.
**Root cause**: Traction rubric caps revenue startups too low. $50K MRR + 15% MoM = traction 14, but should be 16-18. Defensibility scored 12 due to competitive pressure (Flutterwave, Paystack).
**Fix in `strategist.ts`**:
- Update traction rubric 16-20 tier: `"16-20: Revenue with strong retention, multiple paying customers, OR $10K+ MRR with measurable growth"`
- Lower strong threshold from 80 to 75: `"strong": councilScore >= 75 AND no penalties AND traction >= 14 AND team >= 14 AND defensibility >= 12`
- Update in both full and concise prompts

## Fix 3: BOOTSTRAPPED_PENALIZED (medium priority)
**Problem**: Test 10 (Turkish dentist, $6K, 45 users, 3 clinics) got capitalInsufficient penalty despite demonstrated traction.
**Root cause**: Capital penalty is binary — doesn't consider that bootstrapped founders with traction have proven capital efficiency.
**Fix in `strategist.ts`**:
- Add traction override to capitalInsufficient definition: `"EXCEPTION: If the founder has demonstrated real traction (paying users, active beta users, or MRR) with existing budget, this penalty DOES NOT APPLY — the traction itself proves capital sufficiency for current stage."`
