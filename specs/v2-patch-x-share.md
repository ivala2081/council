## Goal

Make the "Tweet" button generate a viral, engaging tweet instead of a generic one-liner.

**Current state:** The tweet button opens Twitter intent with text like:
`"Council verdict: PIVOT — AI tool for contract analysis"` + hashtag-less URL.

**After this patch:** The tweet will look like:
```
Council said PIVOT on my startup idea.

"AI tool that reads legal contracts for freelancers"

Why: 12 competitors exist and none own the niche — Council says narrow fast.

#Council #StartupIdea
```
...with the shareable verdict card (OG image) auto-attached by Twitter.

**Why this matters:** The current tweet is bland. No one retweets "Council verdict: GO". People DO retweet "Council roasted my startup idea and honestly… it's right." The tweet text is the difference between 0 shares and organic distribution.

**What this patch does NOT do:** UI redesign, new pages, new API routes, database changes, new dependencies.

### 2. How the Tweet Flow Works (existing)

```
User clicks Tweet
  → handleTweet() in verdict-card.tsx
  → reads verdict.shareable?.tweet (Claude-generated) OR uses fallback string
  → opens: https://twitter.com/intent/tweet?text=[encoded text]&url=[encoded verdictUrl]
  → Twitter appends the URL to the tweet, auto-fetches OG image from /v/[id]
  → user sees pre-filled tweet with verdict card preview
```

**Key insight:** The URL is appended SEPARATELY by Twitter intent (`&url=` param). Twitter counts URLs as exactly 23 characters regardless of length. So tweet text budget = 280 - 23 - 1 (space) = **256 chars max**.

### 3. Two Levers for Tweet Quality

**Lever 1 (primary): Update the schema description** so Claude generates better tweet text at verdict creation time. This is the right fix — the tweet should be generated by the AI that has full context about the verdict.

**Lever 2 (safety net): Update `handleTweet` fallback** to produce a better string when `shareable.tweet` is missing or lacks hashtags.

Both levers must be implemented. The schema fix ensures new verdicts get great tweets. The client-side fix ensures hashtags are always present (in case the model doesn't follow the schema description exactly).

## Implementation Steps

### Step 1: Update `prompts/v2-output-schema.json` — Tweet Field Description

Find the `tweet` property (around line 162):

```json
"tweet": {
  "type": "string",
  "maxLength": 280,
  "description": "Pre-written tweet about this verdict"
}
```

Replace with:

```json
"tweet": {
  "type": "string",
  "maxLength": 230,
  "description": "Pre-written viral tweet about this verdict. REQUIRED FORMAT (4 lines, use \\n for newlines):\nLine 1: 'Council said [VERDICT] on my startup idea.' (VERDICT = GO / PIVOT / DON'T)\nLine 2: (blank)\nLine 3: One brutal insight from the verdict — the sharpest reason, max 100 chars, no hedging, no jargon\nLine 4: (blank)\nLine 5: '#Council #StartupIdea'\nTotal must be under 230 chars. URL will be appended separately — do NOT include a URL. Do NOT add quotes around Line 1. Keep Line 3 punchy and shareable — write it like something worth retweeting."
}
```

### Step 2: Update `src/components/verdict-card.tsx` — handleTweet Function

Find the `handleTweet` function (around line 133):

```typescript
const handleTweet = () => {
  const text = verdict.shareable?.tweet ?? `Council verdict: ${verdict.verdict} — ${verdict.idea_summary}`;
  const url = verdictId ? `${window.location.origin}/v/${verdictId}` : "";
  const tweetUrl = url
    ? `https://twitter.com/intent/tweet?text=${encodeURIComponent(text)}&url=${encodeURIComponent(url)}`
    : `https://twitter.com/intent/tweet?text=${encodeURIComponent(text)}`;
  window.open(tweetUrl, "_blank");
};
```

Replace with:

```typescript
const handleTweet = () => {
  const verdictLabel = verdict.verdict === "DONT" ? "DON'T" : verdict.verdict;
  const topReason = verdict.reasons[0]?.text ?? "";
  const fallbackText = `Council said ${verdictLabel} on my startup idea.\n\n${topReason.length > 100 ? topReason.slice(0, 97) + "..." : topReason}\n\n#Council #StartupIdea`;

  let text = verdict.shareable?.tweet ?? fallbackText;

  // Safety net: ensure hashtags are always present
  if (!text.includes("#Council")) {
    text = text.trimEnd() + "\n\n#Council #StartupIdea";
  }

  // Enforce 230 char limit (leaves room for URL Twitter appends)
  if (text.length > 230) {
    text = text.slice(0, 227) + "...";
  }

  const url = verdictId ? `${window.location.origin}/v/${verdictId}` : "";
  const tweetUrl = url
    ? `https://twitter.com/intent/tweet?text=${encodeURIComponent(text)}&url=${encodeURIComponent(url)}`
    : `https://twitter.com/intent/tweet?text=${encodeURIComponent(text)}`;
  window.open(tweetUrl, "_blank");
};
```

## Why This Approach

| Option | Pros | Cons |
|---|---|---|
| Only update schema description | Clean — AI generates the best text with full verdict context | Model might not always follow format |
| Only update client-side fallback | Always guaranteed | Fallback loses verdict nuance, generic |
| **Both (this spec)** | AI generates quality text, client ensures hashtags always present | Minimal extra complexity |

The `tweet` field in the schema description is the primary lever. The client-side safety net is 5 lines of code — no reason not to include it.
